# How to get your Google Client ID

1.  **Go to the Google Cloud Console**: [https://console.cloud.google.com/](https://console.cloud.google.com/)
2.  **Create a New Project**:
    - Click the project dropdown in the top-left and select "New Project".
    - Give it a name like `EduGen AI`.
3.  **Configure OAuth Consent Screen**:
    - Search for "OAuth consent screen" in the search bar.
    - Choose **External** and click **Create**.
    - Fill in the required app information (App name, User support email, Developer contact info).
    - **CRITICAL:** **DO NOT** attempt to add `localhost` to the **"Authorized domains"** section at the bottom. Google will reject it. Localhost is already whitelisted for development. Leave "Authorized domains" completely BLANK.
    - Click **Save and Continue** until you finish.
4.  **Create Credentials**:
    - Go to the **Credentials** tab on the left.
    - Click **+ CREATE CREDENTIALS** at the top and select **OAuth client ID**.
    - Select **Web application** as the Application type.
    - Under **Authorized JavaScript origins**, click **+ ADD URI** and add:
        - `http://localhost`
        - `http://localhost:5173` (Must match your React port exactly, NO trailing slash!)
    - **Authorized redirect URIs**: (Usually not needed for this pop-up flow, but you can add the same URLs).
    - Click **Create**.
5.  **Copy the Client ID**:
    - A popup will show your **Client ID** (it ends in `.apps.googleusercontent.com`).
    - Copy this and paste it into `GOOGLE_CLIENT_ID` in your `.env` files.

> [!TIP]
> Make sure to click **"Publish App"** on the OAuth consent screen tab if you want people other than yourself to be able to log in!

## Note on JWT Security
I have also added a `JWT_SECRET_KEY` in your backend `.env`. In production, this should be a long, random string (e.g., generated by `openssl rand -hex 32`). This key ensures that your user sessions cannot be tampered with.
